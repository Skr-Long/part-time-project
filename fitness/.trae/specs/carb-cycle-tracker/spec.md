# 碳循环记录系统 - Product Requirement Document

## Overview
- **Summary**: 一个基于 Next.js 14 的碳循环饮食记录全栈 Web 应用，支持用户注册登录、初始数据记录、宏量营养素计算（百分比模式和体重模式）、一周循环安排和打卡功能。
- **Purpose**: 帮助用户科学地进行碳循环饮食管理，通过精准的宏量营养素计算和便捷的打卡系统，实现减脂、增肌或维持体重的目标。
- **Target Users**: 健身爱好者、需要精准控制饮食的人群、进行碳循环饮食的人

## Goals
- 提供完整的用户注册与登录系统
- 支持用户记录基础数据（体重、身高、年龄、性别、目标）
- 实现两种宏量营养素计算模式：百分比模式和体重模式
- 提供一周碳循环安排（默认经典 7 天 + 自定义）
- 实现极简打卡功能（带可选详细记录）
- UI 简洁美观，响应式设计

## Non-Goals (Out of Scope)
- 不支持多用户协作或社交功能
- 不包含食物数据库和营养成分查询
- 不支持支付或订阅功能
- 不包含运动记录功能
- 不提供移动端原生应用（仅响应式 Web）

## Background & Context
碳循环是一种流行的饮食策略，通过周期性地调整碳水化合物的摄入量，来优化身体代谢、促进脂肪燃烧或肌肉增长。本系统旨在为进行碳循环饮食的用户提供一个完整的管理工具，从数据录入、计划制定到每日打卡，形成完整的使用闭环。

技术栈选择：
- **Next.js 14**: 全栈一体化框架，支持服务端组件和客户端组件，提供优秀的开发体验和性能
- **PostgreSQL + Prisma**: 稳定的关系型数据库搭配类型安全的 ORM
- **shadcn/ui + Tailwind CSS**: 构建简洁美观的现代化 UI
- **NextAuth.js**: 与 Next.js 深度集成的认证方案

## Functional Requirements
- **FR-1**: 用户注册与登录 - 用户可以使用邮箱和密码注册新账户，登录已注册账户
- **FR-2**: 初始数据记录 - 用户首次登录后需填写体重、身高、年龄、性别、目标（减脂/增肌/维持）、活动水平
- **FR-3**: 营养计划设置 - 支持百分比模式和体重模式两种宏量营养素计算方式，用户可自定义参数
- **FR-4**: 宏量营养素计算 - 根据用户数据和计算模式，自动计算 BMR、TDEE 以及每日目标宏量营养素
- **FR-5**: 一周循环安排 - 提供默认经典 7 天方案，支持用户自定义调整每天的碳日类型（高/中/低）
- **FR-6**: 极简打卡 - 用户可以快速完成当日打卡，标记为已完成
- **FR-7**: 可选详细记录 - 用户可以选择录入实际摄入的宏量营养素数值和备注
- **FR-8**: 打卡历史查看 - 用户可以查看历史打卡记录和完成状态

## Non-Functional Requirements
- **NFR-1**: UI 简洁美观 - 使用现代化设计语言，卡片式布局，柔和阴影，清新的配色方案
- **NFR-2**: 响应式设计 - 完美适配桌面、平板和手机设备
- **NFR-3**: 性能 - 页面加载时间 < 2 秒，API 响应时间 < 500ms
- **NFR-4**: 安全性 - 密码使用 bcrypt 加密存储，使用 HTTPS，防止 SQL 注入
- **NFR-5**: 可维护性 - 代码结构清晰，模块化，TypeScript 类型覆盖完整

## Constraints
- **Technical**:
  - 必须使用 Next.js 14 作为主框架
  - 必须使用 PostgreSQL 数据库
  - 必须使用 TypeScript
  - UI 必须使用 Tailwind CSS 和 shadcn/ui 组件库
- **Business**:
  - 项目需要在合理时间内完成（预计 2-3 周）
  - 优先完成核心功能，次要功能可迭代优化
- **Dependencies**:
  - Prisma ORM
  - NextAuth.js
  - bcryptjs (密码加密)
  - zod (表单验证)

## Assumptions
- 用户了解碳循环饮食的基本概念
- 用户有基本的网络和 Web 应用使用经验
- PostgreSQL 数据库服务可用
- 部署环境支持 Node.js 18+

## Acceptance Criteria

### AC-1: 用户注册功能
- **Given**: 用户在注册页面
- **When**: 用户输入有效的邮箱、密码（至少 6 位）并确认密码
- **Then**: 系统创建新用户账户，自动登录并跳转到初始数据设置页面
- **Verification**: `programmatic`
- **Notes**: 邮箱必须唯一，密码需加密存储

### AC-2: 用户登录功能
- **Given**: 用户已注册，在登录页面
- **When**: 用户输入正确的邮箱和密码
- **Then**: 系统验证成功，创建 Session，根据用户状态跳转到对应页面
- **Verification**: `programmatic`

### AC-3: 初始数据设置
- **Given**: 用户首次登录
- **When**: 用户填写体重、身高、年龄、性别、目标、活动水平并提交
- **Then**: 数据保存成功，显示计算结果预览，用户可以进入仪表盘
- **Verification**: `programmatic`

### AC-4: 百分比模式计算
- **Given**: 用户已设置基础数据，选择百分比模式
- **When**: 系统根据用户设置的百分比（碳水、蛋白质、脂肪）
- **Then**: 正确计算出高/中/低三种碳日的每日宏量营养素目标（克数）
- **Verification**: `programmatic`
- **Notes**: 验证公式：碳水热量=总热量×碳水比例，碳水克数=碳水热量/4

### AC-5: 体重模式计算
- **Given**: 用户已设置基础数据，选择体重模式
- **When**: 系统根据用户设置的克数/公斤体重
- **Then**: 正确计算出高/中/低三种碳日的每日宏量营养素目标（克数）
- **Verification**: `programmatic`
- **Notes**: 验证公式：碳水克数=体重×每公斤碳水克数

### AC-6: 一周循环安排
- **Given**: 用户登录系统
- **When**: 用户查看或修改一周循环安排
- **Then**: 默认显示经典 7 天方案，用户可以修改任意一天的类型，修改后保存生效
- **Verification**: `programmatic`

### AC-7: 极简打卡
- **Given**: 用户在打卡页面或仪表盘
- **When**: 用户点击"完成打卡"按钮
- **Then**: 当天打卡状态标记为已完成，更新打卡历史
- **Verification**: `programmatic`

### AC-8: 可选详细记录
- **Given**: 用户在打卡页面
- **When**: 用户展开详细记录，输入实际摄入的碳水、蛋白质、脂肪克数
- **Then**: 数据保存成功，与打卡记录关联
- **Verification**: `programmatic`

### AC-9: UI 简洁美观
- **Given**: 用户使用应用
- **When**: 用户浏览各个页面
- **Then**: 界面设计简洁、现代、美观，配色协调，符合设计规范
- **Verification**: `human-judgment`

### AC-10: 响应式布局
- **Given**: 用户使用不同设备
- **When**: 用户在桌面、平板、手机上访问
- **Then**: 界面正确适配，元素布局合理，无溢出或重叠
- **Verification**: `human-judgment`
